# Setup Upstash Redis

Vercord uses Upstash Redis for rate limiting to prevent abuse. Rate limiting is **optional** and can be enabled or disabled via the `ENABLE_RATE_LIMITING` environment variable.

<Callout type="error">
  **Security Warning**: Disabling rate limiting removes protection against abuse and DDoS attacks. Only disable rate limiting if you understand the security implications and have alternative protection mechanisms in place (e.g., Vercel's built-in rate limiting, Cloudflare, or other DDoS protection). Without rate limiting, your webhook endpoint is vulnerable to being overwhelmed by malicious or excessive requests.
</Callout>

<Steps>
<Step>

### Create an Upstash Account

Go to [console.upstash.com](https://console.upstash.com/) and sign up or log in.

</Step>
<Step>

### Create a Redis Database

Click **Create Database** and configure:

- **Name**: Anything (e.g., `vercord`)
- **Region**: Choose the region closest to your Vercel deployment (usually `us-east-1` for Vercel)
- **Type**: Regional is fine

</Step>
<Step>

### Copy the Credentials

In your database dashboard, find the **REST API** section and copy:

- `UPSTASH_REDIS_REST_URL`
- `UPSTASH_REDIS_REST_TOKEN`

<Callout type="warn">
  Save these values â€” you'll need them for the environment variables.
</Callout>

</Step>
</Steps>
