# Environment Variables

<Steps>
<Step>

### Open Your Versend Project Settings

Go to your Versend deployment in Vercel → **Settings** → **Environment Variables**.

</Step>
<Step>

### Add the Required Variable

<TypeTable
  type={{
    WEBHOOK_INTEGRATION_SECRET: {
      description: 'The signing secret from Vercel webhook',
      type: 'string',
      required: true,
    },
  }}
/>

</Step>
<Step>

### Add Provider Variables

Add the variables for at least one provider. See the [Providers](/docs/providers) section for details.

**Discord:**

<TypeTable
  type={{
    DISCORD_WEBHOOK_URL: {
      description: 'Discord webhook URL',
      type: 'string',
    },
    DISCORD_WEBHOOK_USERNAME: {
      description: 'Custom bot username',
      type: 'string',
      default: 'Versend',
    },
    DISCORD_WEBHOOK_AVATAR_URL: {
      description: 'Custom bot avatar URL',
      type: 'string',
    },
  }}
/>

**Telegram:**

<TypeTable
  type={{
    TELEGRAM_BOT_TOKEN: {
      description: 'Bot token from @BotFather',
      type: 'string',
    },
    TELEGRAM_CHAT_ID: {
      description: 'Chat/group/channel ID',
      type: 'string',
    },
  }}
/>

</Step>
<Step>

### Add Rate Limiting (Optional)

<TypeTable
  type={{
    UPSTASH_REDIS_REST_URL: {
      description: 'Upstash Redis REST URL',
      type: 'string',
    },
    UPSTASH_REDIS_REST_TOKEN: {
      description: 'Upstash Redis REST token',
      type: 'string',
    },
  }}
/>

<Callout type="warn">
  Rate limiting is enabled automatically when both Upstash variables are set. See [Setup Upstash](/docs/installation/setup-upstash) for details.
</Callout>

</Step>
<Step>

### Redeploy

Go to **Deployments** and click the three dots on the latest deployment →
**Redeploy**.

</Step>
</Steps>

## Done!

Your Versend instance is now ready. Trigger a deployment on any monitored
project to test the integration.
